# 偏见防御机制（Meta-Cognitive Layer）

STORM 原始论文明确指出系统不自检，来源偏见和事实错配会悄悄潜入。本 skill 通过 4 层防御解决。

---

## Layer 1: 来源红队（Source Red-Teaming）

对每个主导性叙事，强制召唤一个"Adversary"视角，其唯一任务是找到最强的反面论据。

### Adversary Prompt

```
你是一名严格的真相核查员。以下是一个关于"{{TOPIC}}"的主流观点：

[主流观点]: {{DOMINANT_NARRATIVE}}
[支持者]: {{SOURCE_A}}, {{SOURCE_B}}

你的任务：
1. 找到这个叙事中**最容易被攻击的 3 个弱点**
2. 对每个弱点，提供**具体可验证的反例或数据**
3. 评估主流观点是否**故意忽略了某些关键事实**
4. 给出你的置信度评分（0.0-1.0），说明这个叙事有多可靠

输出格式：
- 弱点 1: [描述] → 反例: [具体数据/事件] → 严重程度: [高/中/低]
- 弱点 2: ...
- 弱点 3: ...
- 综合判断: [一句话]
- 置信度: X.X
```

**规则**: Adversary 的输出必须被 Phase 3 的综合简报完整收录，不得忽略或降级。

---

## Layer 2: 置信度地板（Confidence Floor）

### 评分标准

| 分数 | 含义 | 处理方式 |
|------|------|----------|
| 0.90–1.00 | 高度确信 | 可作为事实陈述 |
| 0.75–0.89 | 较有信心 | 可作为主要论据，但需标注条件 |
| 0.60–0.74 | 中等信心 | 仅作为假设/可能性提出 |
| 0.40–0.59 | 低信心 | 降级为 "有争议的说法"，需另寻佐证 |
| <0.40 | 高度不确定 | **不得纳入综合简报**，仅列入 "未解决问题" |

### 标注规范

每个断言必须携带双标签：
```markdown
[来源:原始URL或文档名] [置信度:X.X] [时效:YYYY-MM]
```

示例：
```markdown
LLM 在代码生成任务上的准确率已超过 80%。
[来源:https://arxiv.org/abs/2401.xxxxx] [置信度:0.92] [时效:2024-01]
```

## Layer 3: Wiki 冲突浮现（Wiki Contradiction Surfacing）

在 Phase 3 综合之前，**强制检索**已有 Wiki 概念：

```python
# 自动化检查链
search_files(pattern="topic_keyword", path="/Wiki/concepts/")
→ read_file(matching_concepts)
→ diff(新结论 vs 已有结论)
→ if 冲突:
    → 在综合简报中开设 [⚠️ 与现有知识冲突] 章节
    → 不覆盖旧概念，而是链接到它并标注分歧点
```

**规则**: 静默覆盖已有知识是**禁止的**。所有冲突必须显性呈现。

---

## Layer 4: 时效衰减标记（Temporal Decay Tagging）

每个事实断言携带 `[时效:YYYY-MM]` 标签。当 skill 被重新调用且检测到旧研究时：

```python
if today - tag_date > 3_months:
    → 触发自动重扫（Phase 1 限定范围更新）
    → 新数据产生 "增量更新" 而非 "全新重写"
```

这保证 Wiki 中的概念不会变成"僵尸知识"。

## 偏见预警信号

当检测到以下模式时，skill 必须自动标记并提示用户：

| 模式 | 信号 | 应对措施 |
|------|------|----------|
| 回音室 | 所有来源来自同一圈层/语言 | 强制跨语言搜索 |
| 近因效应 | >80% 来源来自最近 30 天 | 检索历史背景和长期趋势 |
| 权威崇拜 | 仅凭大牌名字断言，无实质证据 | 降级为 "名人观点"，降权重 |
| 幸存者偏差 | 只谈成功案例 | 强制要求失败案例分析 |
| 动机归因 | 用"因为他们想赚钱"解释一切 | 引入经济学家视角但不独尊 |
| 数据缺失 | 关键数据点标注为 "N/A" 或 "未公开" | 透明度标记，不编造数字 |

---

## 防御层激活顺序

```
Phase 1 扫描 ──► Layer 1 (红队) 在生成每张卡片后自动运行
            ──► Layer 2 (置信度) 每张卡片必须自评

Phase 2 矛盾 ──► Layer 3 (Wiki冲突) 在生成矛盾地图前运行

Phase 3 综合 ──► 如果任何断言置信度 <0.70 → 触发 Layer 1 重审
            ──► 标注 Layer 4 时效标签

Phase 4 评审 ──► 完整 4 层复核 → 生成质量分数
```
